# 长期投资基金日报设计

## 目标

本自动化用于在每个工作日收集半导体与黄金相关信息，并生成适合长期投资者每天阅读的执行型日报。

当前数据模型为 `long_term_review_v4`，不兼容旧逻辑。

报告每天只回答四个问题：

- 长期配置逻辑是否改变。
- 今天每只基金到底买入、卖出、持有、观望，还是不操作。
- 如果买入或卖出，本次建议金额是多少。
- 支撑这次动作的真实证据是什么。

报告不是新闻列表，也不是盘中喊单。

固定覆盖基金：

- `012552 天弘芯片产业ETF联接A`：半导体核心配置。
- `021532 天弘半导体设备指数A`：半导体设备弹性仓。
- `000218 国泰黄金ETF联接A`：黄金防守仓，后续黄金新增只加到这只基金。

组合存量但不输出行动卡：

- `002611 博时黄金ETF联接A`：存量黄金仓，计入黄金总防守仓，不再建议加仓。

不再固定覆盖：

- `006533`

凡是报告中提到基金代码，必须同时带基金名称。

## 用户组合上下文

报告必须结合当前组合，不允许只看单只基金涨跌。

当前组合输入：

- 总资产约 `160000` 元。
- `012552 天弘芯片产业ETF联接A` 当前约 `11500` 元。
- `021532 天弘半导体设备指数A` 当前约 `4400` 元。
- `000218 国泰黄金ETF联接A` 当前约 `11100` 元。
- `002611 博时黄金ETF联接A` 当前约 `9600` 元，只保留，不加仓。
- 剩余约 `123400` 元在债基和货币基金。

组合计算要求：

- 半导体合计 = `012552 天弘芯片产业ETF联接A + 021532 天弘半导体设备指数A`。
- 黄金合计 = `000218 国泰黄金ETF联接A + 002611 博时黄金ETF联接A`。
- 债基/货币基金 = 总资产 - 半导体合计 - 黄金合计。
- `002611 博时黄金ETF联接A` 不进入实时执行链路，不输出行动卡，但必须影响黄金是否需要补仓的判断。
- 如果黄金总防守仓比例已经不低，`000218 国泰黄金ETF联接A` 不应因为自身单只金额偏低而自动补仓。

## 核心原则

### 用户动作优先

行动卡只允许展示用户可直接理解的动作：

- `买入`
- `卖出`
- `持有`
- `观望`
- `今日不操作`

旧行动词不再展示，也不再作为 `data.json` 主字段输出：

- `按原计划执行`
- `半额执行`
- `小额靠近`
- `暂停新增`
- `仅观察`
- `今日不输出执行建议`

如果内部实现仍需要节奏判断，只能作为推导过程使用，不能出现在报告主文案中。

### 金额必须来自当次信息

报告必须给出实际金额建议，不能只写“可买”“可卖”。

不得设置固定默认买入规则，例如“总资产 0.5%”。

金额建议必须由当次报告信息动态推导：

- 当前基金金额和占总资产比例。
- 所属资产组仓位，例如半导体合计和黄金合计。
- 盘中估算和底层代理是否显示追高、回落或失真。
- 正式净值的近 1 日、近 1 周、近 1 月表现。
- 最近 `1-2` 个月原始信息是否支持长期逻辑。
- 该基金角色：核心仓、弹性仓、防守仓、存量仓。
- 是否存在未确认申购缺口。

行动卡可以给三个金额档位：

- `谨慎档`：适合不想承受更多波动的人。
- `均衡档`：本次报告认为更匹配当前证据和仓位的金额。
- `积极档`：只有长期逻辑强、仓位仍有空间、当日不追高时才给出。

必须同时给出 `recommended_amount`，即本次更建议执行的具体金额。

若动作是 `持有`、`观望` 或 `今日不操作`，金额必须为 `0`。

若动作是 `卖出`，金额用负数记录，例如 `-2000`，并说明卖出理由。卖出只在长期逻辑明显转弱、仓位显著超出约束或风险证据明确时出现。

### 证据支撑必须贴近动作

每张行动卡必须有 `evidence_support`，用于解释为什么是这个动作和金额。

证据可以来自：

- 原始长期信息，例如 WSTS、World Gold Council、SAFE 等。
- 仓位证据，例如半导体合计占比、黄金合计占比。
- 行情证据，例如盘中估算、底层代理、近 1 日、近 1 周、近 1 月表现。
- 判断逻辑，例如设备仓不能快速接近核心仓、黄金不是进攻仓。

证据必须服务于本次动作，不能堆砌无关数据。

### 长期逻辑优先

长期逻辑复核必须回答：

- 最近 `1-2` 个月有什么关键原始信息。
- 这些信息是否支持、削弱或改变 `6-24` 个月配置逻辑。
- 实际基金表现是否印证或提醒风险。
- 当前组合仓位是否允许继续执行。
- 因此今天动作和金额如何调整。

实时数据只做执行过滤器，不能单独改变长期逻辑。

### 申购延迟必须进入判断

报告必须提醒基金申购不是即时成交：

- `15:00` 前下单，通常按当日基金净值确认，份额下一交易日或更晚确认。
- `15:00` 后下单，通常顺延到下一交易日净值确认。
- 晚间报告可以继续给执行建议，但必须说明实际承接的是下一交易日净值风险。
- 若今天已有未确认申购，后续报告应避免重复建议同一计划金额。

当前自动化没有接入真实交易记录，因此必须在数据缺口中说明“未确认申购需要人工同步”。

### 正式净值只做复核

正式净值用于确认前一交易日表现和近 1 日、近 1 周、近 1 月结果，不得直接替代当天执行过滤器。

## 运行时间

- 频率：每个工作日。
- 建议时间：上午 `10:15-10:30`，默认时区 `Asia/Shanghai`。
- 也允许晚上手动拉取报告，用来结合当日最新数据决定是否行动。
- 若遇中国内地基金市场休市日，仍可生成报告，但必须标注休市，并且不输出当天执行建议。

## 输出文件

每次运行只覆盖：

```text
reports/
  latest/
    data.json
    report.html
    sources.md
```

不保留日期目录。需要追溯时依赖版本控制历史。

## 数据结构

`data.json` 必须先生成，`report.html` 和 `sources.md` 只能从 `data.json` 渲染。

当前数据模型：

```json
{
  "model_version": "long_term_review_v4",
  "report_date": "2026-06-12",
  "generated_at": "2026-06-12T02:30:00.000Z",
  "data_quality": {},
  "portfolio_context": {},
  "settlement_context": {},
  "top_conclusion": {},
  "thesis_reviews": {},
  "funds": [],
  "source_pool": {},
  "info_source_errors": []
}
```

旧字段 `portfolio_thesis`、`decision_status`、`decision_feed` 不再使用。

旧行动字段 `execution_plan`、`trigger_check` 不再作为主输出使用。

### `top_conclusion`

```json
{
  "status": "green | yellow | red",
  "status_label": "可执行 | 收紧 | 暂停",
  "headline": "今日建议：买入半导体，不新增黄金",
  "action_summary": "012552 天弘芯片产业ETF联接A 买入 800 元；021532 天弘半导体设备指数A 买入 400 元；000218 国泰黄金ETF联接A 持有 0 元",
  "total_buy_amount": 1200,
  "total_sell_amount": 0,
  "focus": ["..."],
  "settlement_warning": "...",
  "data_status": "实时链路完整，可信度 100%"
}
```

顶部结论不得重复展示组合仓位摘要。组合仓位放到页面最后。

### `action_plan`

每只行动卡基金必须包含：

```json
{
  "action": "buy | sell | hold | watch | no_action",
  "action_label": "买入 | 卖出 | 持有 | 观望 | 今日不操作",
  "amount_tiers": [
    { "label": "谨慎档", "amount": 500, "description": "..." },
    { "label": "均衡档", "amount": 800, "description": "..." },
    { "label": "积极档", "amount": 1200, "description": "..." }
  ],
  "recommended_amount": 800,
  "recommended_label": "本次更建议 800 元",
  "how_to_execute": "若 15:00 前下单，通常按当日净值确认；已有未确认申购则跳过。",
  "reason": "...",
  "evidence_support": [],
  "settlement_note": "..."
}
```

行动卡不得包含“什么情况改主意”或“停止条件”。

### `portfolio_context`

```json
{
  "total_assets": 160000,
  "as_of": "user_supplied_static",
  "positions": [],
  "groups": [],
  "cash_like": {},
  "notes": []
}
```

每个 `positions` 条目必须包含：

- 基金代码和基金名称。
- 当前金额和占总资产比例。
- 今日估算涨跌。
- 近 1 日、近 1 周、近 1 月表现。
- 当前状态说明。

## 新鲜度规则

每条实时或准实时数据都必须记录：

- `data_time`
- `fetch_time`
- `freshness_minutes`
- `usable_for_today_execution`
- `source_name`
- `source_url`

默认门槛：

- 基金盘中估算：`10` 个有效交易分钟内。
- 底层实时代理：`5` 个有效交易分钟内。
- 黄金 ETF 或黄金现货代理：`5` 个有效交易分钟内。
- 正式净值不限新鲜度，但只能用于复核。

## 基金规则

### `012552 天弘芯片产业ETF联接A`

定位：半导体核心配置。

- 长期逻辑为 `green` 且实时链路完整时，可以输出 `买入`。
- 当前半导体总仓位约 `9.9%`，仍属于中等偏低配置，核心仓可作为半导体主要新增入口。
- 盘中明显上涨时，金额档位要收紧。
- 盘中小幅回落且长期逻辑未变时，可以提高均衡档，但不能写成重仓信号。
- 长期逻辑为 `yellow` 时，动作应偏 `观望` 或低金额 `买入`。
- 长期逻辑为 `red` 时，动作应为 `今日不操作` 或 `卖出`，取决于风险证据。

### `021532 天弘半导体设备指数A`

定位：半导体设备弹性仓。

- 执行约束必须比 `012552 天弘芯片产业ETF联接A` 更严格。
- 当前设备仓约 `2.8%`，可以保留弹性，但不应快速放大到接近核心仓。
- 设备链上涨明显时，优先降低金额或观望。
- 设备链回落时也只能给小额，因为波动和回撤都更大。
- 只有设备国产化、行业景气或政策逻辑继续增强时，才允许输出小额 `买入`。

### `000218 国泰黄金ETF联接A`

定位：黄金防守仓。

- 黄金不是进攻仓，不因单日下跌自动补仓。
- `000218 国泰黄金ETF联接A` 当前约 `6.9%`，但黄金合计含 `002611 博时黄金ETF联接A` 后约 `12.9%`。
- 后续黄金新增只加到 `000218 国泰黄金ETF联接A`。
- 若黄金总防守仓比例不低，动作应偏 `持有` 或 `观望`，金额为 `0`。
- 只有黄金总防守仓比例不足，且实时数据没有显示极端失真时，才考虑小额 `买入`。
- 若美元、美债收益率或实际利率信号明显压制黄金，动作偏 `观望` 或 `今日不操作`。

## HTML 输出结构

### 第一层：顶部结论

必须包含：

- 抓取时间。
- 数据质量状态和可信度百分比。
- 今日结论。
- 建议买入总金额和卖出总金额。
- 每只行动卡基金的动作和金额。
- 申购确认提醒。

不得包含组合仓位摘要。

### 第二层：今日操作建议

必须展示三张行动卡：

- `012552 天弘芯片产业ETF联接A`
- `021532 天弘半导体设备指数A`
- `000218 国泰黄金ETF联接A`

每张卡必须包含：

- 动作。
- 谨慎档、均衡档、积极档。
- 本次更建议金额。
- 怎么做。
- 原因。
- 证据支撑。
- 申购提醒。

不得包含“停止条件”或“什么情况改主意”。

### 第三层：长期逻辑复核

默认展示：

- 结论。
- 是否改变配置逻辑。
- 当前仓位含义。
- 对今天动作和金额的影响。
- 后续观察项。

默认折叠：

- 最近 `1-2` 个月发生了什么。
- 原始信息证据。
- 结合基金表现怎么看。

### 第四层：正式净值复核

使用表格覆盖三只行动卡基金：

- 基金代码和基金名称。
- 基金角色。
- 最新正式净值。
- 今日估算。
- 近 1 日。
- 近 1 周。
- 近 1 月。
- 复核结论。

正式净值只做复核，不覆盖今日动作。

### 第五层：来源与不确定性

必须说明：

- 实时数据来源。
- 正式净值来源。
- 长期复核证据来源。
- 申购确认延迟。
- 未确认申购缺口。
- 数据缺口。
- 今日金额建议的主要不确定性。

### 第六层：组合仓位概览

放在最后，作为背景材料。

必须展示：

- 总资产估算。
- 半导体合计金额和占比。
- 黄金合计金额和占比，并标注 `002611 博时黄金ETF联接A` 只保留不加仓。
- 债基/货币基金金额和占比。
- 每只组合基金的当前金额、占比、今日估算、近 1 日、近 1 周、近 1 月、当前状态。

## `sources.md`

必须记录：

- 实时执行过滤器来源。
- 正式净值复核来源。
- 长期逻辑复核证据来源。
- 组合仓位来源。
- 申购延迟说明。
- 金额建议说明。
- 数据时间、抓取时间、新鲜度、是否可用于今日执行。
- 可信度备注。

## 写作要求

- 中文撰写。
- 面向非专业投资者，尽量白话。
- 先给结论，再给依据。
- 每段尽量不超过 `120` 个中文字。
- 区分事实、推断和不确定性。
- 不写盘中喊单。
- 不把数据新鲜度写成投资理由。
- 不把长期配置建议和当天动作混为一谈。
- 不把 `000218 国泰黄金ETF联接A` 单只占比和黄金总仓位混淆。
- 晚间下单必须提醒可能按下一交易日净值确认。

## 自动化执行步骤

每次运行必须：

1. 读取本设计文档。
2. 创建或覆盖 `reports/latest/`。
3. 抓取实时执行过滤器数据、正式净值、原始信息证据。
4. 计算新鲜度并生成 `data.json`。
5. 生成 `portfolio_context` 和 `settlement_context`。
6. 生成 `thesis_reviews`，把原始信息、基金表现和组合仓位合并成长期逻辑复核。
7. 生成每只基金的 `action_plan`，包含动作、金额档位、本次建议金额、证据支撑和申购提醒。
8. 生成不重复仓位摘要的 `top_conclusion`。
9. 渲染 `report.html`。
10. 渲染 `sources.md`。
11. 自检三份文件是否一致。

## 自检清单

- 是否只输出 `012552 天弘芯片产业ETF联接A`、`021532 天弘半导体设备指数A`、`000218 国泰黄金ETF联接A` 三张行动卡。
- 是否生成 `model_version: long_term_review_v4`。
- 是否不再依赖 `portfolio_thesis`、`decision_status`、`decision_feed`。
- 是否不再展示旧行动词。
- 是否每张行动卡都有动作、金额档位、本次建议金额、原因、证据支撑和申购提醒。
- 是否不再展示“停止条件”或“什么情况改主意”。
- 是否把行动卡放在第二层。
- 是否把组合仓位概览放在最后。
- 是否所有基金代码都带基金名称。
- 是否把 `002611 博时黄金ETF联接A` 计入黄金总仓位但不输出行动卡。
- 长期逻辑复核是否结论先行、证据折叠。
- 正式净值是否只用于复核。
- 来源与不确定性是否披露组合仓位来源、金额建议说明和未确认申购缺口。
