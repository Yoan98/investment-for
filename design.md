# 基金定时信息收集报告自动化设计

## 目标

本自动化用于在指定时间收集和整理投资判断所需信息，辅助判断半导体基金和黄金基金的买入、持有、减仓或卖出。使用者投资风格偏长期，报告默认服务于长期配置、分批买入、仓位再平衡和风险控制，而不是短线追涨杀跌。报告必须高信息密度、低噪音、可追溯，禁止输出泛泛新闻、营销软文、不可验证传闻和无关个股八卦。

当前关注/持有基金代码：

- `012552`
- `021532`
- `006533`
- `000218`
- `002611`

## 运行时间

- 频率：每个工作日，周一到周五。
- 时间：上午 `10:00`。
- 默认时区：`Asia/Shanghai`。
- 若当天为中国内地基金市场休市日，仍可生成报告，但必须明确标注“休市/无最新净值”，并使用最近可得交易日数据。

## 目录结构

自动化必须在仓库根目录下按以下结构产出文件：

```text
reports/
  YYYY/
    MM/
      YYYY-MM-DD/
        report.html
        sources.md
```

示例：

```text
reports/2026/06/2026-06-03/report.html
reports/2026/06/2026-06-03/sources.md
```

## 文件职责

### `report.html`

主报告文件。必须是完整 HTML 页面或可直接打开的 HTML 片段，包含内联 CSS。页面必须用中文撰写，结构清晰，表格可读，重要结论使用颜色或标签突出。

必须包含：

- 顶部摘要：日期、数据时间、市场环境一句话结论、最重要的 3-5 个判断。
- 持仓基金表：基金代码、基金名称、方向、最新净值/估值或最近可得净值、近 1 日/1 周/1 月表现、关键风险、数据来源。
- 半导体总体分析：当前走势、短期预估、长期预估、关键驱动因素、反向信号。
- 黄金总体分析：当前走势、短期预估、长期预估、关键驱动因素、反向信号。
- 总体操作框架：基于半导体和黄金总体分析，说明当日适合偏进攻、防守、均衡还是观望。
- 单基金操作建议：对每一只基金分别给出买入、加仓、持有、减仓、卖出或观望建议，并明确该建议如何对应到总体分析。
- 今日重点观察清单：只列真正影响判断的事件、数据或价格点位。
- 信息来源摘要：列出主要来源和用途，不堆砌链接。
- 免责声明：说明内容仅用于信息整理和风险判断，不保证收益，不构成确定性投资承诺。

### `sources.md`

来源记录文件。必须记录本次报告使用的关键数据来源，便于复核。

建议格式：

```markdown
# Sources - YYYY-MM-DD

## 基金数据

- 来源名称：
- 链接：
- 用途：
- 数据时间：
- 可信度备注：

## 市场数据

- 来源名称：
- 链接：
- 用途：
- 数据时间：
- 可信度备注：

## 新闻/事件

- 来源名称：
- 链接：
- 用途：
- 发布时间：
- 可信度备注：
```

## 数据优先级

优先使用：

- 基金公司官网、基金公告、基金合同、招募说明书和定期报告。
- 东方财富/天天基金等基金净值与估值数据。
- 交易所、指数公司、央行、统计机构、财政部、美联储等官方或准官方数据。
- 主流财经数据源和可信新闻源。

避免使用：

- 无来源自媒体观点。
- 标题党财经文章。
- 未标注发布时间或来源不明的信息。
- 单一个股传闻。
- 与半导体、黄金、美元、美债、实际利率、央行购金、地缘风险、基金净值无关的信息。

## 判断规则

报告必须区分三类内容：

- 事实：可由数据或来源直接验证。
- 推断：基于事实得出的判断，需要说明逻辑。
- 不确定性：数据不完整、市场未开盘、来源冲突或估值仅为盘中估算时必须标注。

### 长期投资风格规则

报告默认采用长期投资视角，所有结论必须优先回答“是否影响长期配置逻辑”，其次才讨论短期波动。

必须遵循：

- 不因单日涨跌直接给出买入或卖出建议，除非出现重大基本面、政策、流动性或风险约束变化。
- 短期波动主要用于判断分批买入、暂停买入、再平衡或风险提示。
- 长期配置逻辑必须关注 `6-24 个月`，必要时补充 `3-5 年` 的结构性判断。
- 卖出或减仓建议必须更严格，通常只在估值极端、仓位过高、长期逻辑恶化、产品不再匹配方向或需要再平衡时给出。
- 买入或加仓建议应优先采用分批、定投、回撤触发或区间触发，不建议一次性重仓。
- 报告必须区分“长期配置建议”和“短期操作提醒”，避免把短期交易信号误写成长期结论。

### 半导体基金判断要点

总体分析必须观察：

- 国内半导体、芯片、半导体设备相关指数或 ETF 表现。
- 海外半导体 ETF 或指数表现，例如 SOXX、SMH 或同类数据。
- A 股成交量、龙头持续性、板块宽度。
- AI 算力、国产替代、设备材料、出口管制、周期复苏等影响因素。

短期预估范围：`1-4 周`。

长期预估范围：`6-24 个月`。


### 黄金基金判断要点

总体分析必须观察：

- 现货黄金或黄金期货价格。
- 黄金 ETF 或上海金相关数据。
- 美元指数。
- 美债收益率和实际利率方向。
- 央行购金、地缘风险、通胀预期。

短期预估范围：`1-4 周`。

长期预估范围：`6-24 个月`。


## 操作建议规则

建议必须保守、条件化，不能机械喊单。操作建议分为两层：总体操作框架和单基金操作建议。由于使用者偏长期投资，建议应默认以长期持有、分批配置、仓位控制和再平衡为核心。

允许使用的动作：

- 买入
- 加仓
- 持有
- 减仓
- 卖出
- 观望

每条建议必须包含：

- 适用对象。
- 动作。
- 对应的总体分析依据。
- 长期配置逻辑是否变化。
- 短期波动对操作节奏的影响。
- 触发条件。
- 风险级别。
- 仓位思路。
- 关键反向信号。

若证据不足，应输出“观望”或“需要核验”，不得强行给确定性建议。


单基金建议必须覆盖所有基金代码：

- `012552`
- `021532`
- `006533`
- `000218`
- `002611`

如果某只基金无法可靠识别，仍必须保留该基金的建议行，但动作应为“需要核验/观望”，并说明缺口。

## HTML 风格规范

HTML 应使用内联 CSS，便于脱离构建工具直接打开。

建议视觉规则：

- 背景使用浅色，正文高对比。
- 表格必须有表头和边框或分隔线。
- 使用标签突出 `利好`、`警惕`、`风险`、`观察`。
- 数字涨跌应清晰标红/标绿，但不要只依赖颜色表达含义。
- 页面宽度建议限制在 `1000px-1200px`。

## 自动化执行步骤

每次运行必须按以下顺序执行：

1. 读取本设计文档：`automation/morning-briefing-design.md`。
2. 确定当前日期、时区和目标输出目录。
3. 创建当日目录：`reports/YYYY/MM/YYYY-MM-DD/`。
4. 检索并核验基金代码、基金名称、基金类型、净值、估值、近 1 日/1 周/1 月表现。
5. 检索半导体和黄金相关市场数据。
6. 检索必要的宏观和事件信息，只保留影响判断的信息。
7. 生成 `report.html`。
8. 生成 `sources.md`。
9. 最终回复用户时，简要说明生成路径和核心结论，不重复粘贴整份 HTML，除非用户要求。

## 质量门槛

生成报告前必须自检：

- 是否明确标注数据日期和数据时间。
- 是否使用正确基金代码：`012552`、`021532`、`006533`、`000218`、`002611`。
- 是否识别 `006533` 不是纯半导体/黄金基金。
- 是否包含半导体和黄金的总体分析、短期判断和长期判断。
- 是否体现使用者偏长期投资风格。
- 是否区分长期配置建议和短期操作提醒。
- 是否包含每只基金的条件化操作建议。
- 每只基金的操作建议是否明确对应到总体分析。
- 是否剔除了无关新闻和垃圾信息。
- 是否记录了来源。
- 是否写入了两个规定文件。

如果任何关键数据无法获得，报告不能静默跳过；必须在 HTML 和 `sources.md` 中标注缺口。
