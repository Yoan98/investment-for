# 长期投资基金日报设计

## 目标

本自动化用于在每个工作日收集半导体与黄金相关信息，并生成适合长期投资者每天阅读的执行型日报。

报告每天只回答四个问题：

- 最近一两个月的信息和基金表现，是否改变长期配置逻辑。
- 当前组合仓位是否支持继续新增。
- 今天是否适合执行原定计划。
- 如果执行，具体基金应该正常、减半、小额靠近、暂停，还是只观察。

报告不是新闻列表，也不是盘中喊单。

当前固定覆盖基金：

- `012552`：半导体核心配置。
- `021532`：半导体设备弹性仓。
- `000218`：黄金防守仓，后续黄金新增只加到这只基金。

组合存量但不再输出行动卡：

- `002611`：存量黄金仓，计入黄金总防守仓，不再建议加仓。

不再固定覆盖：

- `006533`

## 用户组合上下文

报告必须结合当前组合，不允许只看单只基金涨跌。

当前组合输入：

- 总资产约 `160000` 元。
- `012552` 当前约 `11500` 元。
- `021532` 当前约 `4400` 元。
- `000218` 当前约 `11100` 元。
- `002611` 当前约 `9600` 元，只保留，不加仓。
- 剩余约 `123400` 元在债基和货币基金。

组合计算要求：

- 半导体合计 = `012552 + 021532`。
- 黄金合计 = `000218 + 002611`。
- 债基/货币基金 = 总资产 - 上述权益和黄金仓。
- `002611` 不进入实时执行链路，不输出基金行动卡，但必须影响黄金是否需要补仓的判断。
- 如果黄金总防守仓比例已经不低，`000218` 不应因为自身单只金额偏低而自动补仓。

## 核心原则

### 长期逻辑优先

长期逻辑复核必须先回答：

- 最近 `1-2` 个月有什么关键原始信息。
- 这些信息是否支持、削弱或改变 `6-24` 个月配置逻辑。
- 实际基金表现是否印证或提醒风险。
- 当前组合仓位是否允许继续执行。
- 因此今天的执行节奏如何调整。

### 结论先行，证据折叠

长期逻辑复核必须把结论和证据分开。

默认展示：

- 结论。
- 是否改变配置逻辑。
- 对当前仓位的含义。
- 对今天执行节奏的影响。
- 后续观察项。

默认折叠：

- 最近 `1-2` 个月发生了什么。
- 原始信息证据。
- 结合基金表现怎么看。

这样报告可以先扫结论，需要复盘时再展开证据。

### 原始信息必须进入复核，不单独堆放

不单独渲染“近期信息 / 长期信息”页面区块。

原始信息必须并入对应方向的长期逻辑复核，作为证据支撑。每条证据必须包含：

- 来源。
- 日期。
- 原始事实。
- 链接。
- 对长期逻辑的含义。

信息只有在支撑判断时才进入主报告。

### 实时数据只是执行过滤器

基金盘中估算、底层代理和黄金 ETF 行情只回答：

- 今天执行原计划是否追高。
- 今天下跌是否只是波动，还是需要暂停。
- 是否需要减半、小额靠近或仅观察。

实时数据不得单独改变长期逻辑。

### 申购延迟必须进入判断

报告必须提醒基金申购不是即时成交：

- `15:00` 前下单，通常按当日基金净值确认，份额下一交易日或更晚确认。
- `15:00` 后下单，通常顺延到下一交易日净值确认。
- 晚间报告可以继续给执行建议，但必须说明实际承接的是下一交易日净值风险。
- 若今天已有未确认申购，后续报告应避免重复建议同一计划金额。

当前自动化没有接入真实交易记录，因此必须在数据缺口中说明“未确认申购需要人工同步”。

### 正式净值只做复核

正式净值用于确认前一交易日表现和近 1 日 / 近 1 周 / 近 1 月结果，不得直接替代当天执行过滤器。

## 运行时间

- 频率：每个工作日。
- 建议时间：上午 `10:15-10:30`，默认时区 `Asia/Shanghai`。
- 也允许晚上手动拉取报告，用来结合当日最新数据决定是否行动。
- 若遇中国内地基金市场休市日，仍可生成报告，但必须标注休市，并且不输出当天执行建议。

## 输出文件

每次运行只覆盖：

```text
reports/
  latest/
    data.json
    report.html
    sources.md
```

不保留日期目录。需要追溯时依赖版本控制历史。

## 数据结构

`data.json` 必须先生成，`report.html` 和 `sources.md` 只能从 `data.json` 渲染。

当前数据模型：

```json
{
  "model_version": "long_term_review_v3",
  "report_date": "2026-06-12",
  "generated_at": "2026-06-12T02:30:00.000Z",
  "data_quality": {},
  "portfolio_context": {},
  "settlement_context": {},
  "top_conclusion": {},
  "thesis_reviews": {},
  "funds": [],
  "source_pool": {},
  "info_source_errors": []
}
```

旧字段 `portfolio_thesis`、`decision_status`、`decision_feed` 不再使用。

### `portfolio_context`

```json
{
  "total_assets": 160000,
  "as_of": "user_supplied_static",
  "positions": [],
  "groups": [],
  "cash_like": {},
  "notes": []
}
```

必须包含每个组合分组的金额和占比：

- 半导体合计。
- 黄金合计，包含 `000218` 和存量 `002611`。
- 债基/货币基金。

### `settlement_context`

```json
{
  "generated_at_local": "2026-06-12 18:00:00",
  "order_cutoff_time": "15:00",
  "is_after_cutoff": true,
  "execution_timing_label": "若现在下单，通常按下一交易日净值确认",
  "notes": []
}
```

必须说明申购确认延迟和未确认申购缺口。

### `data_quality`

```json
{
  "status": "complete | partial | blocked",
  "label": "实时执行链路完整",
  "confidence_pct": 100,
  "complete_fund_count": 3,
  "fund_count": 3,
  "oldest_realtime_freshness_minutes": 3.8,
  "can_issue_today_execution": true,
  "reason": "..."
}
```

### `top_conclusion`

```json
{
  "status": "green | yellow | red",
  "status_label": "长期逻辑继续成立",
  "headline": "长期逻辑未变，今天只调整执行节奏",
  "action_summary": "012552 按原计划执行；021532 半额执行；000218 仅观察",
  "portfolio_summary": "半导体约 9.9%，黄金合计约 12.9%，现金/债基约 77.1%",
  "reason": "..."
}
```

### `thesis_reviews`

必须包含 `semiconductor` 和 `gold` 两个对象。

```json
{
  "semiconductor": {
    "title": "半导体长期逻辑",
    "status": "green | yellow | red",
    "status_label": "继续成立",
    "conclusion": "...",
    "review_window": "最近 1-2 个月",
    "recent_developments": ["..."],
    "evidence_items": [],
    "fund_evidence": [],
    "allocation_impact": "...",
    "portfolio_impact": "...",
    "execution_impact": "...",
    "watch_items": []
  }
}
```

`evidence_items` 必须使用真实原始信息，不允许用基金估值涨跌凑数。

`fund_evidence` 必须使用实际基金数据，不允许泛泛写“走势较强/较弱”。

## 新鲜度规则

每条实时或准实时数据都必须记录：

- `data_time`
- `fetch_time`
- `freshness_minutes`
- `usable_for_today_execution`
- `source_name`
- `source_url`

默认门槛：

- 基金盘中估算：`10` 个有效交易分钟内。
- 底层实时代理：`5` 个有效交易分钟内。
- 黄金 ETF 或黄金现货代理：`5` 个有效交易分钟内。
- 正式净值不限新鲜度，但只能用于复核。

## 基金规则

### `012552`

定位：半导体核心配置。

- 长期逻辑为 `green` 且实时链路完整时，允许按原计划执行。
- 当前半导体总仓位约 `9.9%`，仍属于中等偏低配置，核心仓可作为半导体主要新增入口。
- 盘中小幅回落且长期逻辑未变时，可以小额靠近。
- 盘中上涨明显时，不追高；可正常执行或减半执行。
- 长期逻辑为 `yellow` 时，只允许减半或观察。
- 长期逻辑为 `red` 时，暂停新增。

### `021532`

定位：半导体设备弹性仓。

- 执行约束必须比 `012552` 更严格。
- 当前设备仓约 `2.8%`，可以保留弹性，但不应快速放大到接近核心仓。
- 设备链上涨明显时，优先提示不追高和减半。
- 设备链回落时也只能小额，因为波动和回撤都更大。
- 只有设备国产化、行业景气或政策逻辑继续增强时，才允许维持计划内小额执行。

### `000218`

定位：黄金防守仓。

- 黄金不是进攻仓，不因单日下跌自动补仓。
- `000218` 当前约 `6.9%`，但黄金合计含 `002611` 后约 `12.9%`。
- 后续黄金新增只加到 `000218`，`002611` 只作为存量计入总黄金仓位。
- 若长期防守逻辑未变，优先维持组合比例。
- 只有黄金总防守仓比例不足，且实时数据没有显示极端失真时，才考虑小额补足。
- 若美元、美债收益率或实际利率信号明显压制黄金，动作偏暂停或观察。

## HTML 输出结构

### 第一层：顶部结论

必须包含：

- 抓取时间。
- 数据质量状态和可信度百分比。
- 长期逻辑总状态。
- 组合仓位摘要。
- 今日动作总建议。
- 一句话原因。

### 第二层：组合仓位概览

必须展示：

- 总资产估算。
- 半导体合计金额和占比。
- 黄金合计金额和占比，并标注 `002611` 只保留不加仓。
- 债基/货币基金金额和占比。
- 组合对今日动作的约束。

### 第三层：长期逻辑复核

这是报告的核心层，必须替代旧的“原始信息层”。

每个方向默认展示：

- 结论。
- 是否改变配置逻辑。
- 当前组合仓位含义。
- 对今天执行节奏的影响。
- 后续观察项。

每个方向默认折叠展示：

- 最近 `1-2` 个月发生了什么。
- 原始信息证据。
- 结合基金表现怎么看。

证据必须直接展示来源、日期、事实和链接。

### 第四层：具体基金行动卡

每只基金必须一张行动卡，包含：

- 今日建议。
- 执行幅度。
- 节奏。
- 原因。
- 仓位约束。
- 申购延迟提示。
- 证据。
- 停止条件。

建议动作只能使用：

- `按原计划执行`
- `半额执行`
- `小额靠近`
- `暂停新增`
- `仅观察`
- `今日不输出执行建议`

### 第五层：正式净值复核

使用表格覆盖三只基金：

- 基金代码。
- 基金名称。
- 基金角色。
- 最新正式净值。
- 近 1 日。
- 近 1 周。
- 近 1 月。
- 复核结论。

正式净值只做复核，不覆盖今日执行结论。

### 第六层：来源与不确定性

必须说明：

- 实时数据来源。
- 正式净值来源。
- 长期复核证据来源。
- 组合仓位来源。
- 申购确认延迟。
- 数据缺口。
- 今日建议的主要不确定性。

## `sources.md`

必须记录：

- 实时执行过滤器来源。
- 正式净值复核来源。
- 长期逻辑复核证据来源。
- 组合仓位来源。
- 申购延迟说明。
- 数据时间、抓取时间、新鲜度、是否可用于今日执行。
- 可信度备注。

## 写作要求

- 中文撰写。
- 面向非专业投资者，尽量白话。
- 先给结论，再给依据。
- 每段尽量不超过 `120` 个中文字。
- 区分事实、推断和不确定性。
- 不写盘中喊单。
- 不把数据新鲜度写成投资理由。
- 不把长期配置建议和当天执行节奏混为一谈。
- 不把 `000218` 单只占比和黄金总仓位混淆。
- 晚间下单必须提醒可能按下一交易日净值确认。

## 自动化执行步骤

每次运行必须：

1. 读取本设计文档。
2. 创建或覆盖 `reports/latest/`。
3. 抓取实时执行过滤器数据、正式净值、原始信息证据。
4. 计算新鲜度并生成 `data.json`。
5. 生成 `portfolio_context` 和 `settlement_context`。
6. 生成 `thesis_reviews`，把原始信息、基金表现和组合仓位合并成长期逻辑复核。
7. 生成每只基金的今日执行计划、仓位约束、申购延迟提示和停止条件。
8. 生成 `top_conclusion`。
9. 渲染 `report.html`。
10. 渲染 `sources.md`。
11. 自检三份文件是否一致。

## 自检清单

- 是否只输出 `012552`、`021532`、`000218` 三张行动卡。
- 是否生成 `model_version: long_term_review_v3`。
- 是否不再依赖 `portfolio_thesis`、`decision_status`、`decision_feed`。
- 是否不再渲染“今日执行信号汇总”。
- 是否不再单独渲染原始信息层。
- 是否展示组合仓位概览。
- 是否把 `002611` 计入黄金总仓位但不输出行动卡。
- 长期逻辑复核是否结论先行、证据折叠。
- 长期逻辑复核是否包含原始信息证据。
- 长期逻辑复核是否包含实际基金表现证据。
- 每只基金是否有执行幅度、仓位约束、申购延迟提示和停止条件。
- 正式净值是否只用于复核。
- 来源与不确定性是否披露组合仓位来源和未确认申购缺口。
